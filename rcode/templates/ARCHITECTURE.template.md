## Data Flow

### Request Flow

```
User Action → Client Component → API Route → Business Logic → Database
     ↑              ↓                ↓              ↓            ↓
     └──── UI Update ◀── Response ◀── Validation ◀── Query Result
```

### Authentication Flow

```
1. User submits credentials
2. [Auth provider] validates
3. JWT/Session token issued
4. Token stored in [httpOnly cookie / localStorage]
5. Subsequent requests include token
6. API middleware validates token
7. User context available in route handlers
```

### Data Mutation Flow

```
1. User action triggers form submission
2. Client-side validation (immediate feedback)
3. API route receives request
4. Server-side validation (authoritative)
5. Database transaction
6. Optimistic UI update (if applicable)
7. Success/error response
8. UI reconciliation
```
